# Session 2026-04-22 — CRM Glass для Turbo Performance

## Контекст

Заказчик: «Турбо Перформанс» (dev.turbo-performance.ru) — новый клиент, отдельный от Twinr. Попросил: взять стилевой язык Liquid Glass, изученный на twinr-liquid-glass-standalone.html, и воссоздать **1-в-1 визуально** новый проект с другим содержимым — систему управления контентом (CMS) для рекламных кампаний.

Входные данные:
- Продакшн CMS открыта в Chrome MCP (логин Татьяна)
- Структура из 4 разделов: Проекты · Организации · Документы · Библиотека
- Реальные данные: Москва 2025 (проект), Василек/Рога и копыта/РМГ Сервис (организации), 14 радио/digital ресурсов
- Бекдроп: тот же sunset (Matterhorn) что у Twinr

## Решения (почему так)

**1. Backdrop тот же (sunset Matterhorn) — а не свой.**
Заказчик сказал «1-в-1 стиль». Стилевой язык = типография + токены + backdrop. Новый backdrop для другого проекта размывает узнаваемость. Если захочет свой — поменяем одной строкой.

**2. Activity feed вместо hub-плиток на Главной.**
В первой версии были 4 плитки (Проекты/Организации/Документы/Библиотека) = дублировали sidebar и KPI. Админ Татьяна открывает CRM чтобы узнать **что произошло** — activity feed отвечает на этот вопрос, плитки-дубликаты не отвечают.

**3. Действия → в начало таблицы (после чекбокса).**
Изначально было справа (как в оригинальной CMS). Но с 17 колонками Действия уходили за viewport, требовали горизонтальный скролл. Перенёс в начало — всегда видно, не требует скролла. UX побеждает pattern-matching.

**4. Kebab-меню вместо 4 inline-кнопок.**
3 кнопки (Создание/Файлы/Справки) редкие, одна (Редактировать) частая. Оставил Редактировать как primary-icon, остальные убрал в kebab ⋮ — стало компактнее, действий больше (+Дублировать/+Удалить).

**5. `overflow-y: clip` вместо `visible` на `.t-wrap`.**
CSS-специфика: `overflow-x: auto` автоматически включает `overflow-y: auto`. Это создавало невидимый вертикальный scroll-контекст → sticky thead залипал в середине первой строки (первую строку было видно ПОД thead). `clip` убивает Y-скролл, но сохраняет X.

**6. `:has()` для scope правил по таблицам с чекбоксами.**
Правило `td:first-child { width: 40px }` было глобальным — ломало все остальные таблицы (первая колонка сжималась до 40px, текст «Опция первая» переносился). Через `:has(tbody input.row-checkbox)` применяется только к таблицам с чекбоксами. Fallback через ID для старых браузеров.

**7. Vertical-align: middle на всех td.**
В строках с wrapped-текстом (наименование услуги = 5 строк) чекбоксы и иконки Действий при vertical-align: top уплывали к верху, а визуально данные справа — посередине. Middle выровнял всё на одну оптическую линию.

**8. Глобальный `.cell-link` стиль.**
Первая версия scope-ила стиль только под `.t-table` → ссылки в .req-value на детали проекта падали на browser-default blue. Заказчик сказал «ссылки некрасивые». Глобализовал: белый текст + тонкое coral-подчёркивание, на hover coral.

## Блокеры / компромиссы

- **Responsive**: работает на desktop (1440px+). На 1280 и ниже 17-колоночная таблица требует горизонтальный скролл. Mobile/tablet не оптимизирован (CRM-020).
- **Inter Font**: подгружается с fonts.googleapis.com. Без интернета — system fallback (SF Pro/Segoe UI). Для полной offline-работы надо embedить в base64 (~200 KB), пока не сделал.
- **JS-routing локальный**: всё через location.hash. При next.js реализации маршрутизацию перепишем на app router.
- **Реальные данные прибиты**: все контрагенты и ресурсы из dev.turbo-performance.ru захардкожены в HTML. Для интеграции с API потребуется шаблонизация.
